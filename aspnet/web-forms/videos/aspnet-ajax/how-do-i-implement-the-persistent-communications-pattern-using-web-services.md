<a name="how-do-i-implement-the-persistent-communications-pattern-using-web-services"></a>[Gewusst wie:] Implementiert die permanente Kommunikation-Muster, die mithilfe von Webdiensten?
====================
durch [Joe Stagner](https://github.com/JoeStagner)

In einer herkömmlichen Website im Browser und dem Server eine laufende Kommunikation nicht beibehalten, aber kommunizieren nur als Reaktion auf der Benutzer eine Aktion ausführt. In einer modernen-Website, wo die Seite für einen Anwendungscontainer wird, kann es vorteilhaft sein für Browser und dem Server eine laufende Kommunikation verwaltet werden, sodass Seitenupdates auftreten können, ohne dass der Benutzer eine Aktion aus. Dies wird als permanente Kommunikation Muster für AJAX bezeichnet. ASP.NET AJAX bietet zwei verschiedene Arten für Webentwickler, die permanenten Kommunikation Muster zu implementieren. In einer früheren Video wurde erläutert, wie die ASP.NET AJAX-UpdatePanel als Grundlage für die Implementierung zu verwenden. In diesem Video lernen wir zum Implementieren von demselben Musters, die über einen JavaScrpt-Aufruf an einen Webdienst, der wodurch die Notwendigkeit eines ASP.NET AJAX-UpdatePanel entfernt.
