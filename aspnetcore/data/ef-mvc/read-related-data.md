### <a name="performance-considerations"></a><span data-ttu-id="2422f-137">Überlegungen zur Leistung</span><span class="sxs-lookup"><span data-stu-id="2422f-137">Performance considerations</span></span>

<span data-ttu-id="2422f-138">Wenn Sie, die Sie für jede Entität abgerufen verknüpfte Daten benötigen wissen, bietet eager loading, häufig die beste Leistung, da eine einzelne Abfrage, die an die Datenbank gesendet, in der Regel effizienter als separate Abfragen für jede Entität abgerufen wird.</span><span class="sxs-lookup"><span data-stu-id="2422f-138">If you know you need related data for every entity retrieved, eager loading often offers the best performance, because a single query sent to the database is typically more efficient than separate queries for each entity retrieved.</span></span> <span data-ttu-id="2422f-139">Nehmen wir beispielsweise an, dass jede Abteilung zehn Verwandte Kurse gelten.</span><span class="sxs-lookup"><span data-stu-id="2422f-139">For example, suppose that each department has ten related courses.</span></span> <span data-ttu-id="2422f-140">Unverzüglichem Laden von alle verknüpften Daten führt in einer einzelnen (Join)-Abfrage und einem einzelnen Roundtrip-mit der Datenbank.</span><span class="sxs-lookup"><span data-stu-id="2422f-140">Eager loading of all related data would result in just a single (join) query and a single round trip to the database.</span></span> <span data-ttu-id="2422f-141">Für Kurse für jede Abteilung eine eigene Abfrage führt zu elf Roundtrips zur Datenbank.</span><span class="sxs-lookup"><span data-stu-id="2422f-141">A separate query for courses for each department would result in eleven round trips to the database.</span></span> <span data-ttu-id="2422f-142">Zusätzliche Roundtrips zur Datenbank sind vor allem auf die Leistung beeinträchtigen, wenn Latenz hoch ist.</span><span class="sxs-lookup"><span data-stu-id="2422f-142">The extra round trips to the database are especially detrimental to performance when latency is high.</span></span>

<span data-ttu-id="2422f-143">Andererseits, in einigen Szenarien eine eigene Abfrage ist jedoch effizienter.</span><span class="sxs-lookup"><span data-stu-id="2422f-143">On the other hand, in some scenarios separate queries is more efficient.</span></span> <span data-ttu-id="2422f-144">Unverzüglichem Laden von alle verknüpften Daten in einer Abfrage könnte eine sehr komplexe Verknüpfung generiert werden, verursachen, die SQL Server nicht effizient verarbeiten kann.</span><span class="sxs-lookup"><span data-stu-id="2422f-144">Eager loading of all related data in one query might cause a very complex join to be generated, which SQL Server can't process efficiently.</span></span> <span data-ttu-id="2422f-145">Oder Sie müssen eine Entität Navigationseigenschaften nur für einen Teil einer Reihe von Entitäten zugreifen, die Sie verarbeiten, kann eine eigene Abfrage möglicherweise besser ausgeführt werden, da unverzüglichem Laden aller Elemente im Vorfeld mehr Daten als benötigt abrufen würde.</span><span class="sxs-lookup"><span data-stu-id="2422f-145">Or if you need to access an entity's navigation properties only for a subset of a set of the entities you're processing, separate queries might perform better because eager loading of everything up front would retrieve more data than you need.</span></span> <span data-ttu-id="2422f-146">Wenn die Leistung kritisch ist, empfiehlt es sich zum Testen von Leistung beides Möglichkeiten, um die beste Wahl vornehmen zu können.</span><span class="sxs-lookup"><span data-stu-id="2422f-146">If performance is critical, it's best to test performance both ways in order to make the best choice.</span></span>
